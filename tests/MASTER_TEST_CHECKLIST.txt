================================================================================
ROLE HIERARCHY
================================================================================

1. SYSTEM ADMIN (isSystemAdmin: true)
   - Highest level access
   - Can manage all tenants
   - Can see data across all tenants
   - Can create/edit/delete tenants
   - Test User: admin@speccon.co.za

2. TENANT ADMIN (role: admin, within tenant)
   - Full access within their tenant
   - Cannot see other tenants
   - Cannot access tenant management
   - Can manage users within tenant

3. MANAGER (role: manager)
   - Can see all clients in tenant
   - Can manage users
   - Can view reports
   - Cannot access sensitive admin functions

4. SALESPERSON (role: salesperson)
   - Can only see own clients
   - Cannot view reports
   - Cannot access admin functions
   - Basic CRM functionality only

5. ACCOUNTANT (role: accountant)
   - Financial focus
   - Can edit financial data
   - View-only for most client data
   - Can manage financial year settings

================================================================================
TESTING PROCEDURE
================================================================================

1. Create test users for each role in Firebase Authentication
2. Assign appropriate roles in Firestore users collection
3. Assign users to tenant (tenantId field)
4. Set isSystemAdmin: true for system admin user
5. Login as each user and complete their checklist
6. Document any issues found
7. Verify fixes and retest
